Introduction

이 페이지는 아파치 톰캣 서블릿/JSP 컨테이너에 대한 문서 번들의 최상위 엔트리 지점입니다. 아파치 톰캣 버전 7.0은 자바 커뮤니티 프로세스에서 나온 서블릿 3.0 및 JSP(JavaServer Pages) 2.2 사양을 구현했고, 웹 응용 프로그램 및 웹 서비스의 개발 및 배포를 위한 유용한 플랫폼으로 만드는 많은 추가 기능을 제공합니다.

탐색 메뉴에서 링크 중 하나를 (왼쪽)를 선택하면 더 자세한 설명서가 나옵니다. 사용 가능한 각 사용 설명서는 아래에 자세히 설명되어 있습니다.

아파치 톰캣 User Guide

다음 문서는 아파치 톰캣 7을 다운로드하고 설치하고, 많은 아파치 톰캣 기능을 사용하는 데 도움이 될 것입니다

  1. 소개 - 간략하고, 전반적인 아파치 톰캣 개요.
  2. 설치 - 다양한 플랫폼에 아파치 톰캣을 설치하고 실행하는 방법.
  3. 첫번째 웹 애플리케이션 - 서블릿 스펙에 정의된 웹 애플리케이션의 컨셉에 대한 소개. 웹 애플리케이션 소스 트리의 기본 구조와 웹 애플리케이션 아카이브의 구조, 웹 애플리케이션 배포 설명자 소개를 커버합니다.(/WEB-INF/web.xml).
  4. 디플로이어(Deployer) - 웹 애플리케이션의 배포, 프리컴파일, 검증하기 위한 아파치 톰캣 디플로이어 동작법.
  5. 매니저 - 아파치 톰캣이 실행되는 동안 웹앱 배포, 삭제, 재배포하기 위한 매니저의 동작법.
  6. 렐름(Realm)과 접근 제어 - 컨테이너 관리 보안을 이용하는 웹 애플리케이션에서 사용할 렐름(사용자, 비밀번호, 관련된 역할의 데이터베이스)을 설정하는 방법을 설명.
  7. 보안 관리자 - 웹 애플리케이션의 동작을 세세하게 제어하는 자바 보안 관리자를 설정하고 사용하는 방법.
  8. JNDI 자원(Resources) - 각 웹 애플리케이션별로 제공되는 JNDI 네이밍 컨텍스트에서 표준과 사용자 자원을 설정하는 법.
  9. JDBC 데이터소스 - DB 커넥션 풀과 JNDI 데이터소스 설정하기. 많은 인기있는 데이터베이스에 대한 예제.
  10. 클래스로딩 - 아파치 톰캣에서 클래스 로딩에 대한 정보, 애플리케이션 클래스들이 어디에 위치해야 인식하는지를 포함.
  11. JSP - JSP 컴파일러 사용법과 자스퍼 설정에 대한 정보.
  12. SSL - https 프로토콜을 사용하는 요청을 처리하기 위해 SSL 지원을 설치하고 설정하기.
  13. SSI - 아파치 톰캣에서 서버측 인클루드 사용하기.
  14. CGI - 아파치 톰캣에서 CGI 사용하기.
  15. 프록시 지원 - 프록시 서버 (또는 프록시 서버 기능을 하는 웹 서버) 뒤에서 실행하는 아파치 톰캣 설정하기.
  16. MBean Descriptor - Configuring MBean descriptors files for custom components.
  17. Default Servlet - Configuring the default servlet and customizing directory listings.
  18. 아파치 톰캣 Clustering - Enable session replication in a 아파치 톰캣 environment.
  19. Balancer - Configuring, using, and extending the load balancer application.
  20. Connectors - Connectors available in 아파치 톰캣, and native web server integration.
  21. Monitoring and Management - Enabling JMX Remote support, and using tools to monitor and manage 아파치 톰캣.
  22. Logging - Configuring logging in 아파치 톰캣.
  23. Apache Portable Runtime - Using APR to provide superior performance, scalability and better integration with native server technologies.
  24. Virtual Hosting - Configuring virtual hosting in 아파치 톰캣.
  25. Advanced IO - Extensions available over regular, blocking IO.
  26. Additional Components - Obtaining additional, optional components.
  27. Using Tomcat libraries with Maven - Obtaining Tomcat jars through Maven.
  28. Security Considerations - Options to consider when securing an 아파치 톰캣 installation.
  29. Windows Service - Running Tomcat as a service on Microsoft Windows.
  30. Windows Authentication - Configuring Tomcat to use integrated Windows authentication.
  31. High Concurrency JDBC Pool - Configuring Tomcat to use an alternative JDBC pool.
  32. WebSocket support - Developing WebSocket applications for 아파치 톰캣.
Reference

The following documents are aimed at System Administrators who are responsible for installing, configuring, and operating an 아파치 톰캣 server.

  • Release notes - Known issues in this 아파치 톰캣 release.
  • 아파치 톰캣 Server Configuration Reference - Reference manual that documents all available elements and attributes that may be placed into the 아파치 톰캣 conf/server.xml file.
  • JK Documentation - Complete documentation and HOWTOs on the JK native webserver connector, used to interface 아파치 톰캣 with servers like Apache HTTPd, IIS and others.
  • Servlet 3.0 Specification and Javadoc
  • JSP 2.2 and EL 2.2 Specifications, JSP 2.2 Javadoc (note it is labelled as JSP 2.1), EL 2.2 Javadoc

  • Building from Source - Details the steps necessary to download 아파치 톰캣 source code (and the other packages that it depends on), and build a binary distribution from those sources.
  • Changelog - Details the changes made to 아파치 톰캣.
  • Status - 아파치 톰캣 development status.
  • Developers - List of active 아파치 톰캣 contributors.
  • Functional Specifications - Requirements specifications for features of the Catalina servlet container portion of 아파치 톰캣.
  • Javadocs - Javadoc API documentation for 아파치 톰캣's internals.
  • 아파치 톰캣 Architecture - Documentation of the 아파치 톰캣 Server Architecture.
